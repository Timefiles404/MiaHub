MiaEco — 参数化程序化森林

框架版本 通过规定区域与树种，自动依地形/坡度/方块/水参数选点，并异步并行地把每棵树程序化生长为形态各异、大小不一的树；可随时间推进森林演替。

快速上手

# 1. 站在一角，设置选区两个对角点
/miaeco pos1
（走到对角）/miaeco pos2

# 2. 用选区建立一片森林
/miaeco forest create 云杉林

# 3. 添加树种（使用 config.yml 的默认参数，可在 forests.yml 细调）
/miaeco species add 云杉林 spruce

# 4. 参数化选点 + 种下树苗（异步）
/miaeco plant 云杉林

# 5. 推进时间，观察长大 / 枯死 / 倒伏
/miaeco advance 云杉林 3
/miaeco advance 云杉林 12

0.25.0 河岸融入自然 · 大厅控制台 · 草稿全生命周期

河流彻底融入环境：草坪旁的河，两岸就是草坪——只有中间一条细水路。①低地下切调浅（平原溪流水面只低于草皮 ~2 格，山涧仍深切）；②河岸的沙/砾/卵石镶边整体废除：滩皮只保留宽缓大河贴水线的零星沙洲斑与河心洲，小河两岸一律是所在群系的原生地表；③湍流"滚石岸"删除（水下卵石河床保留）；④岸边植被照旧——芦苇贴水而生，草皮河岸添蕨与矮草，只有植物、没有换皮。

大厅控制台（GUI）：出生平台的主控制台（右键讲台）一屏总览所有世界与草稿——种子/尺寸/比例尺/状态/森林数全在悬浮说明里，点进条目即可 传送/刷新沙盘/续跑/重生成/删除（危险操作 8 秒内二次确认）；生成配置页可直接切换 河流密度/洞穴/崖蚀/地貌 开关，写回 config.yml 并热生效。每块沙盘旁还有自己的参数操作台：草稿的 比例尺/海平面/边缘/竖向缩放/种子 全部可调（世界大小除外），抽卡/水系预览/送产/删除一键可达。

大厅性能重做：进大厅不再卡顿/overload——沙盘采样按区块去重后经全局队列限流（每 tick 至多 4 组区块在途，逐世界串行）；采样结果快照持久化，地形没变的世界零区块加载秒开重建；雪盘铺设分批进行（每 tick 3 行）。

草稿全生命周期：草稿送产后自动存档——删除由草稿创建的世界，草稿即恢复为可编辑形态（雪面保留送产时的样子，seed/参数俱在，改改再送产）；非草稿出身的世界删除时大厅沙盘区直接清空释放。

沙盘与预览画独立可调：hub new 新增 preview=（12~48）设定沙盘边长，与地图实际大小解耦；水系俯视图升级为拼接地图画墙（默认 4×4=512² 像素，控制台 1~6 可调），画墙移到平台东侧外并抬高——不再被雪柱遮挡。

石林奇观下线：柱簇观感僵硬，暂撤出 geo 命令与 terra 自动散布（裸峰改偶发孤石阵），待按真实石林参考做精细化采样后回归（见路线图）。

0.24.1 热修 · 告别"高架桥河"——河流沉进大地

根治河流浮在地表、两岸垫沙砾墙的"高架桥/长城"观感。根源是河流水位来自粗规划场，与精细推理的实际地形局部差几格：旧逻辑在水位高于地形处把 3 格窄岸硬抬到水位兜水，皮肤又是与环境无关的沙砾。现在——

① 默认往下挖：规划水位整体下切 1.5 格偏置（叠加原有侵蚀谷噪声），河流系统性低于地表，在大地上刻出真正的河谷；② 谷壁削坡放宽：高岸的河谷侧壁从 3 格陡坎放宽到高差 ×1.6（最多 14 格）——河沉在有坡度感的河谷里，陡壁自然露岩；③ 抬堤改河漫滩：确需高于地形处不再垫窄堤，而是宽羽化的平缓河漫滩（宽度=抬升量 ×2.5，最多 14 格），地表保持所在群系的正常草地皮肤；④ 滩皮收窄：沙/泥/砾滩材质只保留贴水线 1~2 格一圈，其余河岸全部还给周边群系；⑤ 湖泊保留上限 12→40：被丢弃的洼地曾让过路河"高架"跨越——真湖远比渡槽自然，湖缘掩码外低地也加了平滑羽化。离线校验新增 ±6 格失配模拟与"窄堤墙"断言（抬升列朝外单步落差 ≤2）。

0.24.0 大厅沙盘 · 抽卡造图 · 水力几何 · 全服进度

MiaEco 大厅（/miaeco hub）：一个无限大的专用虚空大厅，每个受管世界一块 20×20 积雪沙盘——高度用雪块+雪层表现（1 层 = 1/8 格精度），水面铺浅蓝玻璃，最高峰/最低谷悬浮字标注；正在生成的世界会随分片完成实时"长出来"，状态牌同步显示阶段与百分比。

抽卡与手修造新图：hub new 开草稿沙盘 → hub roll 抽种子（coarse 粗扫秒级铺盘，无限抽）→ 直接堆/铲雪修形（1 层雪 ≈ 45 米，海平面=3 格雪）→ hub water 预览水系（滴水粒子沿雪面画出河流湖泊 + 沙盘旁画框挂俯视地图画）→ hub confirm 把雪面差量作为草图低频混入扩散地形送入生产——细节仍由模型生成，宏观形态跟随你的雪面（草图持久化，regen/续跑同样生效）。

河流水力几何：起流面积调低（95²→58² 格）——源头密度约 2.7 倍，涓流多而细、逐级树状汇流，水系终于像真实水域图。并按节点坡度做流速建模：陡段窄而深（山涧深切、卵石/苔石河床、岸边滚石），平缓段宽而浅（平原大河可达 ~20 格宽仅 2~3 格深，泥沙河床、泥沙滩涂岸）；河内生态按流速分带——缓流海草丰茂、睡莲点缀、岸边芦苇丛生，湍流河床冲刷裸净；泉眼圈新增湿生杜鹃。

氛围小河流废弃：0.10 时代"每个生态区各刻一条"的小河观感粗糙，terra 自动生态现已全面停用（含画布模式）——河流一律出自全局水文规划。手动 forest atmo feature river=… 不受影响。

进度全服广播：地形任务的聊天进度与 BossBar 现在面向全服所有在线玩家（并留控制台日志），发起者退出不再中断播报；中途进服的玩家自动补挂 BossBar 并收到当前阶段与百分比。

0.23.0 真实水文 · 流域建模与河流地貌

河流从"画线"升级成"流域"：整张地图先做水文建模——洼地填充（Priority-Flood）→ 逐格最陡流向 → 汇水量累计。降水往低处聚，汇水面积过阈值处自然成河：泉眼在高地涌出（陡坡源头一汪苔石小潭），涓流顺谷而下，支流树状汇合、越汇越宽（源头 ~2.4 格宽渐长至大河 ~11 格），终点只有三种归宿——入海、入湖、流出图边。封闭洼地自动成湖泊（水位=溢出口下沿，湖岸齐平），湖满则溢，溢出口继续成出流河——湖链、串珠水系都会自然出现。光滑坡面上的流向做了随机化扰动（不会出现平行直线水道），低地大幅蜿蜒、山地贴谷线。

河流地貌全家桶：山口坡折处堆出冲积扇（砾沙锥面 + 辫状干沟 + 零星枯灌）；大河入海口生成三角洲（汊流劈开 + 泥滩 + 齐平泥岛）；平原蜿蜒段旁点缀牛轭湖（弯月静水）；跌差 ≥3 的天然跌水下方掏出跌水潭；宽缓河段中央偶有河心洲（齐平沙洲）；深切段自动成峡谷（河槽加深、两壁裸岩）；宽缓大河两岸低地重标记为沼泽湿地——柳树、红树与浓水氛围沿大河自然铺开。所有地貌与河道一样是全图规划、逐片栅格化，断点续跑一致。

0.22.1 热修 · 平地涡旋河

修复河流追踪在平缓盆地里"原地打转成一团圈"的病理形态：坡向消失时正弦蜿蜒会单边持续转向。现在跟踪航向变化的滑动累计，持续同向打转即判定为内流终点收束（正常蜿蜒左右交替不受影响）。另附赠离线出图工具 gradle :miaeco:riverMap（真实权重渲染 edge=open 地图的河流平面图）。

0.22.0 全局河流 · 海岸带 · 水岸齐平 · 树冠与奇观自然化

河流重做（全局规划）：此前地图里的河是每个生态区各自刻的——短、直、同向平行，非常假。现在改为全图统一规划：生成开始时用粗高程场扫整张地图（秒级），从高地源头以"坡向+惯性+正弦蜿蜒"追踪入海——蜿蜒不打直、彼此拉开间距、撞上别的河即汇流并使下游加宽（源头 ~2.4 格宽渐至河口 ~7 格，深度随宽度），水位严格单调下行（跌差即天然小跌水），然后逐片栅格化进地形。河道有抛物线剖面、浅滩海草、零星睡莲，岸边偶有芦苇。数量可调（terrain.rivers，0=关）。

水岸齐平：老问题"水永远比岸低一格（-_）"根治了。河岸第一圈陆地顶面钳到水位、海岸线贴水的 sea+1 陆列压到海平面——现在是 -- 齐平的岸线，趟水上岸没有台阶；低于水位的河岸自动填成天然堤，绝不漏水。浅滩带用平滑距离场铺沙/砾，不再零零碎碎。

海岸带群系：森林不再直接怼到海边。距海 12 格内按温度/坡度/分段噪声重标记为：椰林沙滩（成簇椰树 2~4 株）、红树滩（浅水红树带架空根与垂根须、泥滩）、砾石滩（卵石堆、浮木）、滨海草甸（草浪、沙丘草、杜鹃矮丛、野花）、雪滩；陡峭处成海岸崖（裸岩+崖肩蕨草——崖顶森林俯瞰海是合法景观）。同段海岸同一类型（~110 格一段），森林在 5~12 格带内散点式渐次退出。

树冠自然化：非平顶大树的冠顶此前太圆太密（光滑椭球壳）——现在上半球逐列随机内蚀 + 顶面成对凸簇，冠面参差松散，轮廓不再是"绿蛋壳"。小树/细干树若树干裸露天空，自动补一小簇冠盖——活树不再秃杆朝天（枯树/倒木不受影响，它们本就该裸）。

森林疏密大跨度：森林区密度噪声重做——96 格大尺度主导 + 幂曲线拉开：密核 ~1.35、疏区 ~0.15（跨度 8×+），叠加大尺度林窗（约 1/6 面积近乎空地）。森林不再从头密到尾。平原节奏同步修复：大平原按 150 格噪声翻出小林班/疏林/草甸镶嵌，大小平原交替，不再连片。

奇观自然化：蘑菇岩改正弦曲干（像树一样带弧度）+ 半径起伏 + 风蚀微孔，岩帽三变体（经典出檐/圆丘/斜板）帽缘参差，檐下用倒置楼梯收边、帽顶零星台阶；石林/孤石整柱微倾 + 表面微孔 + 细顶台阶收头；峰林微倾 + 出檐层理楼梯。全部风格自动配楼梯/台阶材质（冰塔除外）。

地表过渡与新参数：不同群系交界的顶面方块按距离概率互相咬合 1~3 格（42%/22%/12%，轻微不过火）。world create 新增 edge=open（四周不强制为海：地形直通图边断崖式切开，且山体渐进增幅 +35%——要山地图就用它）与 yscale=（0.5~2.5 竖向缩放，山更高耸）。

性能：植被规划全面多核化——树点适宜度并行预打分、简单生态各区并行预规划、地貌奇观各区并行生成（应用按区序，结果与单线程逐位一致）；树木结构生成本就并行。生态阶段吃满 CPU。

0.21.2 配置自动升级

老服务器的 config.yml 会自动补齐新配置项了：Bukkit 只在文件不存在时写默认配置，所以早期装的服务器永远看不到后来新增的开关（比如整个 terrain: 段——GPU 的 device 就在里面）。现在插件启动时自动升级：新键带着完整中文注释补进去，你改过的旧值原位保留，原文件备份为 config.old.yml。更新到本版重启一次，再打开 config.yml 就能看到全部配置项。

0.21.1 热修 · 更新崩溃与 jar 瘦身

修复 /miah update 时服务器 OOM 崩溃：根因是 Paper 运行时加载插件要对整只 jar 做重映射（映射表就吃几百 MB 堆）。现在 jar 声明了 mojang 映射命名空间（本插件纯 Bukkit API）——Paper 直接跳过重映射；同时 ONNX 推理引擎的 native 库移出插件包（jar 12.6MB → 约 5MB），首次使用地形功能时自动下载（CPU 版 93MB，一次性，断点续传）。MiaEco 更新现在标记为需重启生效：/miah update 后请重启服务器——嵌入原生推理引擎的插件热替换有先天风险，重启是最稳的路径。

0.21.0 生态全覆盖 · 超大地图 · GPU 加速

不再有"区域过大跳过生态"：超大的森林/开阔区会被自然切分（噪声扰动的 Voronoi 生长，边界蜿蜒不规则，配合软边树线看不出接缝）成 ~9 万格的块逐一生态化。同时生态版图补齐了此前空白的一大片：温带/暖/寒海洋（海草甸、海带林、海泡菜、海底孤石）、冰原海（冰面压力脊与冰丘）、海滩（浮木、近水甘蔗、零星棕榈——现在会写入原版 beach 群系）、荒漠重做（只有仙人掌、枯灌、枯树剪影，大漠深处可能藏一汪棕榈绿洲）、恶地（枯灌+陶瓦孤石，配 0.20 的蘑菇岩）。这些"简单生态"走轻量散布，不建森林对象、开销极小——整张地图从海底到峰顶尽可能被覆盖。

稀树草原重做：金合欢密度砍到原来约四分之一——真正的"零零散散几棵树"，树木借助新的双尺度密度噪声（12 格小斑块 × 64 格大起伏）自然聚成"树丛 + 大片空地"，区里还常有一汪小池塘。所有森林的疏密分布都因此更自然。

超大地图（size 上限 1024 → 10240）：地图世界改为分片流水线：按比例尺自动切成 480~960 格的片，推理→铺设→地貌→生态逐片推进——内存恒定，10k×10k 也能跑；每片完成即记档，取消/崩溃后 /miaeco terra resume 从下一片续跑。分片间高程/群系逐位一致（离线校验），坡度与滩涂跨片连续。

GPU 加速：terrain.device: gpu（或 auto）现在开箱即用——插件自动下载 ONNX Runtime CUDA 版与 NVIDIA 官方 CUDA 运行库（cudart/cublas/cudnn/cufft，共约 2GB，分段并行+断点续传+校验，免装 CUDA Toolkit），只要机器有 NVIDIA 驱动（CUDA 12.x）。实测 RTX 5070：整批推理 ~12× 于 CPU（640² 原生场 5.1s vs 61.9s；首次装载含内核编译需几分钟，之后有缓存）。不可用时 auto 自动回退 CPU。注意：CPU 与 GPU 的浮点结果有微小差异——同一张地图请用同一设备生成/续跑，跨设备续跑可能在分片交界留下细微接缝。

0.15.0 地表聚落化 · 微生境选点

全部地表地物告别随机散列：河岸带之外的林床植被同样改为"锚点 + 连通生长 + 自疏留隙"的聚落体系——每个锚点长出一片同一物种的连通斑块（边缘不规则、内部留孔隙），锚点之间强制留白；原先的逐格随机散点压到低配作为背景（聚与散共存，花朵总量不减，只是从"撒胡椒"变成"花田"）。

微生境自动选点：植被按所处环境选择群落类型——树脚绕树基长蕨圈、蘑菇窝、浆果丛（樱花树下是花瓣圈）；岩石旁贴着石头长苔藓蕨类环带（干旱主题换草丛枯灌、雪原换雪堆），湿润主题的岩石侧面偶有发光地衣；水畔是高草滩、湿生花丛、垂滴叶群；树荫下是菌窝、阴生花片，偶尔出一圈完整的蘑菇菌圈（fairy ring）；开阔地则是成片的单种花田、草浪、雪堆与杜鹃小丛。巨岩也讲聚落：四成概率带一两块伴石成石群。

可度量的改善：花朵"成群率"从 0.03~0.24 提升到 0.48~0.77，蘑菇成群率从 ≤0.20 提升到 0.47~0.65，蘑菇长在树脚的比例从约七成提升到 85~94%。

0.14.0 山体岩石硬化 · 地表覆盖 v2（还草与苔藓大区）

山体岩石硬化：陡崖与陡峰坡面整片石质化——石头/安山岩/凝灰岩/圆石按低频斑块自然分选成片，湿润主题嵌苔圆石、偶发砾石碎屑窝。山体识别引入窗口起伏量（不再只看单列高差——连绵陡坡每列落差常常只有 1~2 格，必须看一片才能认出"山"），山麓过渡带以石斑+粗土斑点缀渐变。坎高 ≥3 的裸露立面同步走石系地层带（每列有主导石材、逐块轻微扰动，横向读出地层感），矮坎仍是土质根系面。

坡面还草："比较斜的山全是泥土砂土、看着秃"成为历史——中缓坡的裸土斑覆盖率大幅下调，且裸露的泥土/砂土地面主动铺回草方块；土斑从"大面积裸秃"回归"草坡上的点缀"。

苔藓块大区：苔藓块正式进入地表大面积生成池——按主题湿润度（沼泽/雨林最盛，针叶次之，温带少量，雪原/草原无）圈出成片苔藓地，作为"茂密浅层草"铺在缓坡与林下（树冠下更旺、近水更旺）；区内留草色孔隙、边缘以苔藓地毯羽化过渡，不会一刀切成色块。

有效地表感知：地被植物的底材判断现在读"改造后"的真实地表——草苗不会再种到新铺的石面/沙面上（与湿带沙地上长草的老 bug 一并根除）。

簇/聚落体系：岸线地物不再逐格撒随机数——沿水线取间距化锚点，每个锚点生长一个连通的"坨"：岸石露头（2~5 格核心同顶高的整块岩体 + 楼梯/台阶变种裙边收坡，贴岸半浸入水，告别碎石子观感）、灌木丛（同一种叶连成 3~8 格的坨，最多两格悬出水面且必与岸上本体相连——灌木伸到水边的感觉）、成排芦苇丛、栅栏干小松、草花小片。锚点之间留白，聚与散共存。

自疏机制：睡莲田/荷花丛/海泡菜引入"过密强制消失"——同簇内被更强邻居压制的个体不生成，水面留出间隙不结毯；水下海泡菜大幅减量成零星小簇（水生资源竞争，不会一片一大堆）。

荷花重做：大型垂滴叶——茎秆没入水中（含水）、叶头挺出水面（随机朝向），这才是荷叶该有的样子；杜鹃叶彻底退出水面。泡菜含水根修：海泡菜在 MC 里默认含水，写入端现在对所有可含水方块显式双向设置——水面以上的泡菜真正"枯"了。

增补型冲刷：削不动的垂直岸也要平滑——真悬崖坡脚自动嵌楼梯+圆石堆（talus 坡积），树保护带与削坡后仍有落差的水线嵌楼梯脚，用增补方块把垂直面变弧线驳岸；窄急段冲宽更明显，宽缓段沙洲成簇淤出且只淤河道内缘（绝不截断水路，验证器连通分量全绿）。

0.12.0 阶梯跌水 · 水路连通 · 冲刷模拟 · 3D 露天

瀑布重做（模拟而非假想）：彻底移除"悬空水幕"与人造垂直石壁——跌水改为阶梯化：水位步差全程磨到 ≤2（向上游圆化唇部、牺牲底部方块），跌水列只加深床底，水沿床阶自然流下贴合地形。水体区间连通：相邻水列的竖直水体区间若不相交（水在台阶处断裂），自动把高侧床底向下挖穿到低侧水面之下（迭代至稳定）；仅对角相接的断点由较低的公共邻列转为衔接水列。离线验证器按连通分量判定：全主题零硬断裂、零悬空水。

冲刷模拟（真实地形驱动）：不再只处理"生成时假想的河岸"——以全部水体为源 BFS 外扩 3 圈，逐列按真实高度与"水面+距离"目标做松弛削坡（沙滩→沙草→草地过渡），原始地形里紧邻水面的突兀落差同样被冲刷平顺；高差 >5 的真悬崖保留为峡谷（立面衬砌）。窄急段把贴河低岸冲成浅水（河变宽），宽缓段河缘沉积变浅、偶尔淤出露头沙洲（河变窄）。

植株修正：浮水植物只有睡莲——杜鹃叶不再出现在水面（王莲/水葫芦成历史）；荷叶必定挺出水面 1~2 格（玻璃板茎）；行水段照样长河缘芦苇与睡莲（急流/瀑布仍寸草不生）；水面以上的海泡菜一律"枯"态（不含水）。岸边新增不碰水的一坨坨灌木丛（杜鹃/开花杜鹃/橡叶/云杉叶随机成团）与栅栏干小松（云杉栅栏做干、叶做小冠）穿插其间。

3D 露天判定："露天"不再只看柱顶——与邻列高差 ≥2 的原始台阶/坎面，其朝空气的侧面方块（上下都不是空气）也按面材（泥土/砂土/夯泥）成片重铺；修复河边贴生孤立高位水潭的判定漏洞。

0.11.0 河岸质感 · 植株分带 · 瀑布冲刷潭 · 全地表改造

河岸切面衬砌：河道旁裸露的竖直岸壁不再保留原方块——低岸壁换根土质感（砂土/泥土/砾石、偶嵌苔卵石），高于 3 格的峡谷崖壁换石质（石头/圆石/苔圆石），逐层噪声混铺、过渡自然。配合河漫滩缓坡（贴水细沙滩→草地）与驳岸石组/灌木，岸线彻底告别生硬垂直切。

水生植株分带铺设：参考造景教程"岸边→河心"的组合分布——挺水带（近岸浅水：芦苇秆、荷花丛、泡菜浅床）→ 浮水带（睡莲田、水葫芦团、王莲疏散大浮叶）→ 沉水层（海草甸混双层高海草、金鱼藻、海带柱、海菜花、海泡菜床，可与浮水同列叠加）。物种斑块噪声让同种连片成"荡/丛/田"，不再杂乱聚团；密度随主题丰富度大幅拉开（沼泽/雨林豪华、稀树草原稀疏）。湖塘岸线新增岸带植被（芦苇丛/灌木/草花斑块）。湍流不长植物：瀑布/浅滩及其邻域全禁，行水段只有稀疏沉水。

瀑布冲刷潭：跌水底部碗形冲刷潭——受冲刷的石/砾潭底、潭缘一圈不低于水面的湿石边框（低缘自动垫石防溢流）；瀑布崖面与水幕侧壁整体换受冲刷湿石（苔圆石/圆石/石头），不保留原样。水路连通：两格水仅对角相接时，共用侧块中较低的一块自动转水，宁牺牲岸体也不让水路断裂。

全森林地表改造：soil 特征升级为全地表处理——一切露天地面按坡度（陡坡裸出砂土/泥土/夯泥成片、中坡混砂土斑）与湿度（近水湿土/细沙带）成片重铺，叠加主题土壤斑块与大树缠根圈（岩石面暂不做）。修复：草/花偶尔浮在水面的 bug。

0.10.0 真实水文式河流 · 自然驳岸 · 水生植株 · 汀步

河流算法重做（排水链河网）：不再对边硬连——Priority-Flood 洼地分析现在同时记录每格水的排水方向父链，河道即"高地源头沿父链下行"的真实排水路径，数学上保证不被山脊拦腰截断（修复 0.9.0 河道半途断头）。河遇封闭洼地自动壅水成湖（整洼灌注、恰好填满不外流，串珠湖链），越过溢出口自动生成瀑布跌水（竖直水幕+跌水潭）；多个源头的下游自然交汇成支流汇流；遇到无外流的深大湖泊即以之为河口。源头数量随强度 1~4 条，河宽下游渐宽（源头收窄、河口放宽）。

河床整治：告别"水泥平底+杂色散点"——河床按横向泥沙分选成片铺设（中线淤泥/黏土 → 中带砂砾 → 河缘细沙，低频噪声让同材连片过渡、零星卵石点缀）；剖面为抛物线（中线深 2~4、河缘浅 1）；沿程深潭-浅滩交替，浅滩偶有露出水面的溪石（急滩感）。

河漫滩自然驳岸：垂直切岸成为历史——河岸带削成缓坡滩涂（贴水 +1 缓升到外缘 +3；沙/砾滩涂→草地自然过渡；原生高差 >4 的陡壁保留为峡谷崖景）；沿水线布驳岸石组（2~5 块混石半浸游走、顶苔毯、偶叠两层）与岸顶灌木（杜鹃叶团、偶开花）。

三类水生植株（固定植株结构，参考造景教程）： 挺水——荷花丛（淡/深绿玻璃板作茎、水下含水与水融合；顶为杜鹃叶荷叶，出水高差 0~2 错落，偶海泡菜"莲蓬"、开花杜鹃叶"荷花"）、芦苇荡（岸上甘蔗丛+浅水玻璃板秆顶海泡菜穗）； 浮水——睡莲组（错落留白不密排）、王莲（含水杜鹃叶平贴水面成大浮叶、疏散分布）、水葫芦（含水开花杜鹃叶粉花浮团近岸聚生）； 沉水——双层高海草+海带柱（2~4 高，"更高的藻类"）、狐尾藻草甸（海草成片）、金鱼藻（角珊瑚簇，含水恒活）、海菜花（玻璃板长茎自深床直抵水面顶泡菜浮花）、海泡菜床（同格 1~4 颗随机造型，浅水发光）。 全部按水深/流态/离岸距离自动选址（瀑布急滩不长浮水植物），天然浅水湖塘同样铺设；雪原冻结主题自动跳过。

汀步：浅滩窄段自动铺跨溪石列——天然不规则混石（含主题石料）、步距 1~2 疏密错落、横向抖动不成直线、顶面与水面平或高一格、偶覆苔毯，两端各延几块嵌入草坪与地面齐平；列旁点缀一组大石（苔石砖/石砖混搭）。粉红花簇地物升级为同格 1~4 朵随机朝向。

0.9.0 盆地湖 · 月牙塘 · 5 档水系

盆地湖（封闭洼地整体灌水）：河流特征最先做 Priority-Flood 洼地分析——找出"水恰好填满而不会流到外面"的封闭山谷/洼地，整体灌水到溢出位-1：不破坏地形、天然不外流，正是堰塞湖/谷中湖的意象。湖底铺淤泥/黏土/砂砾，浅水水草、深水海带、湖缘苔石岸线与睡莲；被淹没的既有小水面自动融合抬升。巨型浅洼地有水面预算保护，水位自动回落只淹谷底（不会一片汪洋）。

山侧月牙塘：从山体侧面"突出来"的天然平台水塘——选址要求外向 3 格骤降 ≥4 且背靠山体，内部挖 1~2 格灌水；水面是双圆交集之补集的一半（月牙形，凸弧朝崖外，类似无边泳池），池缘硬石+苔石、靠山侧回填草土，崖壁挂藤。找不到合格山肩就不硬放。

河流意愿大幅增强：走廊接受阈值随强度放宽（雨林 riverStrength 0.7→0.85、沼泽 0.65→0.9——这两个主题现在非常容易出河）；强度高时生成多条河道（最多 3 条，共享雕刻集自然汇流）、河槽随强度加宽。湖与河全不成立才退化为小水潭。

density 上限 5、5 档尤其强烈：全部氛围特征强度上限 3→5；river 的 density ≥4.5 触发 fierce 档——盆地湖必然灌注且数量+1、河道数+1、河槽更宽、中线深挖到 3 层、月牙塘+1。/miaeco atmo feature 森林 river 5 即水系全开。

0.8.0 河流与大型地物

河流（氛围最优先特征）：A* 沿低谷走廊自动择线——对两组对边低点各搜一条走廊，取逆流爬升率更低者；没有真正的谷地（率超阈值）就退化为 2~3 个雕刻小水潭，绝不硬穿山。沿线雕出宽 2~4、中线深 2 的河槽（最多动 3~4 层地表），水位单调下行、被迫深挖超限自动截断并以水湾收尾（不突兀截流）；淤泥/黏土/砂砾河床 + 深槽水草 + 苔石岸线，两端椭圆水湾，雪原整面结冰。河水并入湿度场：垂滴叶、苔毯、芦苇会自发聚向河岸。

枯树天然桥：每条河 1~2 座——窄河段上横跨的深色倒木（云杉/深橡原木），苔毯覆背、垂根挂底、偶有菌菇。滚石巨岩：岩石特征 15%（坡脚加倍）升级为 2~4 格径的实心椭球巨石，底层嵌地、阴面披苔，像从山上滚落的。

小路升级为 A* 成本寻路：坡度/水体/冠层惩罚 + 噪声扰动，小径自动沿等高线与谷缘绕行、避水绕树，坡面放楼梯坡道、平段撒踏石。积水新增覆地薄水膜：近水平地铺 level=7 最薄水层（冻结更新），湿地滩涂质感；水潭获得泥/黏土潭底、深潭水草、潭缘卵石。

选点重构：多树种按 适宜度×密度 加权混植（修复了并列评分时首树种垄断、再次 plant 不进新树种的问题）；再次 plant 成为补植——旧树占位、新一轮填空隙；树间距按各自树种 spacing 空间哈希强制（小间距灌木能钻进乔木间隙）；species density 上限放宽到 5，高密度+小 spacing=灌木海。

色调修缮：树体木板/台阶/楼梯一律深色橡木或云杉（浅色木板出戏），预制树同步重映射，砂岩改夯泥；灌木不再上台阶缘；倒伏木每隔数格向下垫残枝抵地，坡地不悬空。

森林氛围（0.7.0）

参数模型（0.6.0 树库规范重制）

0.6.0 依据一座 999 棵建筑师设计树的树库逐树实测重写了整个生成体系：

曲干体系：树干是连续 S 曲线脊线 + 有机截面扫掠——干径按树库实测规律取胞（h<25 中位仅 2 胞，h38-60 才 7 胞），气势靠曲线与低位分导枝而不是粗度；结构一律六面皮木头（带朝向原木只出现在枯木断口等锯断面），大干表面点缀木板补丁。0.7.0 修缮：小型树（非巨型）大幅收直——像素分辨率下细短干经不起弯折，曲率与微抖都随高度衰减，20 格以上大树才有完整曲线；棕榈改为全木弧干（去掉栅栏上段）。

空壳裂片冠：大冠是 1~2 格厚的叶壳分成 2~4 个错位裂片，内部中空、由放射状伞骨辐条从里面撑住；44% 树库树混 2~3 种树叶，本作按调色板通道混叶；冠底垂 1 宽叶帘（丛林藤帘/柳瀑/秋树彩条）；冠面撒草蕨花或雪层绒饰；树脚配苔石堆与草花圃组景。

树种阵容：oak / spruce / birch / jungle / acacia / dark_oak / mangrove（红树，带扎地气根）/ willow 垂柳（瘤结巨座+下瀑冠）/ palm 棕榈（倾倒弧干+放射叶羽）/ banyan 榕树（巨伞+气根帘）/ maple 秋枫、ginkgo 银杏（羊毛+混凝土+陶瓦彩冠、垂缘）/ cherry 樱花 / cypress 柏树 / bush 灌木（高密度可作水岸小片林）/ snowy_spruce 雪杉。

树库预制树(Prefab)：内置从树库直采的 147 棵手工级预制树（含巨型秋树、垂榕、雪花树、巨杉等），/miaeco test stamp random 直接盖印体验，可按族名（oak/spruce/birch/jungle/special/none）抽取。

连续体型 + 巨大化变异：普通树的体型在 0.62~1.45 间连续分布（越大越稀有），上端个体自然拥有 2x2 粗干与板根；另有 ~0.5% 的超大巨木变异（体型 1.9~2.5，橡/丛林/云杉 3x3、深色橡木 4x4 巨干），由种子确定、贯穿生命周期。测试：/miaeco test oak plant giant。

枯亡形态：枯立木保留整棵树的枝干骨架（剥叶、参差断顶、根系保留）；倒伏木在原地留下树桩+根盘，倒干随体型加粗、带残枝与枯叶团，久置腐朽消失。

月度补间：阶段内逐月长高（连续成熟度），每次 advance 都有可见变化。树叶写入时置 persistent，不会自然凋零；藤蔓全部侧向依附、无平铺无悬空。

森林生态：选点只取选区 XZ 自动找地表；适宜度打分同时决定点位与生长活力（好地长得快）；演替时计算光照竞争——被高大邻树遮蔽的树生长减速、重度遮蔽的幼树会枯死，避免树冠交叉；密度由确定性噪声场自动疏密不均，可用 /miaeco forest density 设置相对倍率。

演替：advance 按月推进年龄，活体树随年龄长大，老树随机枯死成枯立木，枯立木随机倒伏，倒木久置后回归林窗。所有随机性由树种子决定，可复现。
